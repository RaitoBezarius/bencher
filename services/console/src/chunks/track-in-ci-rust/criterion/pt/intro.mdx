Agora que você aprendeu [como conduzir benchmarks de código Rust com Criterion][learn criterion], vamos ver como acompanhar esses benchmarks em CI. [Benchmarking Contínuo][continuous benchmarking] é a prática de executar benchmarks em cada conjunto de mudanças para garantir que as alterações não introduzam regressões de desempenho. A maneira mais fácil de implementar o Benchmarking Contínuo com benchmarks Criterion é usar o Bencher.
