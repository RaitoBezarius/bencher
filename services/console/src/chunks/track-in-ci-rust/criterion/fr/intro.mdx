Maintenant que vous avez appris [comment mesurer les performances du code Rust avec Criterion][learn criterion], voyons comment suivre ces benchmarks dans l'intégration continue (CI). [Le Benchmarking Continu][continuous benchmarking] est la pratique consistant à exécuter des benchmarks sur chaque ensemble de modifications pour s'assurer que les changements n'introduisent pas de régressions de performance. La façon la plus simple d'implémenter le Benchmarking Continu avec les benchmarks Criterion est d'utiliser Bencher.
