## Déploiement

Le moyen le plus simple de déployer Bencher Self-Hosted est d'utiliser un triptyque de sous-commandes CLI `bencher` :

- [`bencher up`][bencher up] : Démarre les conteneurs Bencher Self-Hosted.
  Cela ressemble à l'exécution de [`docker compose up`][docker compose up].
- [`bencher logs`][bencher logs] : Affiche la sortie des conteneurs Bencher Self-Hosted, si `bencher up` a été exécuté avec [le drapeau `--detach`][detach flag].
  Cela ressemble à l'exécution de [`docker compose logs`][docker compose logs].
- [`bencher down`][bencher down] : Arrête et supprime les conteneurs Bencher Self-Hosted, si `bencher up` a été exécuté avec [le drapeau `--detach`][detach flag].
  Cela ressemble à l'exécution de [`docker compose down`][docker compose down].

[docker compose up]: https://docs.docker.com/reference/cli/docker/compose/up/
[docker compose logs]: https://docs.docker.com/reference/cli/docker/compose/logs/
[docker compose down]: https://docs.docker.com/reference/cli/docker/compose/down/
