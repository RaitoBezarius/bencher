## Восстановление после сбоя

Чтобы предотвратить потерю данных между резервными копиями, Bencher предлагает встроенное решение для восстановления после сбоя. Эта функция требует активной лицензии [Bencher Plus][pricing].

Поддерживаемые схемы репликации:

- `file`: Репликация в локальный путь к файлу
- `sftp`: Репликация через SFTP
- `s3`: Репликация в любое совместимое с S3 хранилище блобов
- `abs`: Репликация в Azure Blob Storage
- `gcs`: Репликация в Google Cloud Storage

Используйте раздел [`plus.disaster_recovery`][plus disaster recovery] конфигурации API-сервера, чтобы настроить восстановление после сбоя для вашего самостоятельно хостируемого API-сервера Bencher.
