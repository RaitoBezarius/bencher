### ⚖️ Taille de fichier

La sous-commande CLI <code><a href="/fr/docs/explanation/bencher-run/">bencher run</a></code> peut être utilisée pour suivre la taille du fichier (c'est-à-dire la taille binaire) de vos livrables avec l'option `--file-size`. L'option `--file-size` attend un chemin de fichier vers le fichier dont la taille sera mesurée. Sous le capot, <code><a href="/fr/docs/explanation/bencher-run/">bencher run</a></code> sort les résultats en format JSON [Bencher Metric Format (BMF)][bmf]. Il est donc de bonne pratique d'utiliser explicitement [l'adaptateur `json`][json]. Pour plus de détails, voir [comment suivre la taille de fichier][file size].

La Mesure de `taille de fichier` (c'est-à-dire `octets (B)`) est recueillie. Seule la valeur de la taille de fichier (c'est-à-dire `valeur`) est disponible. Ni `valeur_inférieure` ni `valeur_supérieure` ne sont collectées. La Mesure de `taille de fichier` n'est pas créée par défaut pour tous les Projets. Cependant, lorsque vous utilisez l'option `--file-size`, cette Mesure sera automatiquement créée pour votre Projet.
L'option `--file-size` peut être utilisée plusieurs fois pour suivre plusieurs tailles de fichier.
