### `--project <PROJECT>`

<br />

可选：可以设置 `--project` 选项或 `BENCHER_PROJECT` 环境变量为项目的 slug 或 UUID。
如果指定的值是一个 slug 且项目尚不存在，将为您创建该项目。
但是，如果指定的值是 UUID，则项目必须已经存在。
如果同时指定了两者，则 `--project` 选项优先于 `BENCHER_PROJECT` 环境变量。
如果两者都未指定，则会根据以下信息为您生成一个项目 slug：

1. `git` 仓库的父目录名称（如果存在）。
2. `git` 仓库初始提交的 7 位十六进制短哈希（如果存在）。
3. 本地机器的 13 位字母数字指纹（对于支持的操作系统）。

例如，生成的项目 slug 可能如下所示：`project-abc4567-wxyz123456789`

当为您创建新项目时，无论是指定的 slug 还是生成的 slug，项目都必须属于一个组织。
如果用户已通过身份验证，项目将添加到该用户的个人组织下。
如果用户未通过身份验证，项目将在新的 `unclaimed` 组织下创建。
此组织可以从项目的公共性能页面认领，或者在后续调用 `bencher run` 时通过身份验证使用项目 slug 进行认领。
