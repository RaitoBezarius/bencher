### `--project <PROJECT>`

<br />

可选：可以设置 `--project` 选项或 `BENCHER_PROJECT` 环境变量为项目的 slug 或 UUID。
如果指定的值为 slug 而项目尚不存在，则将为您创建该项目。
然而，如果指定的值是 UUID，则项目必须已存在。
如果两者都指定了，则 `--project` 选项优先于 `BENCHER_PROJECT` 环境变量。
如果两者都未指定，则会根据以下内容为您生成一个项目 slug：

1. `git` 仓库的父目录名称（如果可用）。
2. `git` 仓库初始提交的 7 位十六进制短哈希（如果可用）。
3. 本地机器的 13 位字母数字指纹（对于支持的操作系统）。

例如，生成的项目 slug 可能类似于：`project-abc4567-wxyz123456789`

当为您创建新项目时，无论是指定的还是生成的 slug，
该项目必须属于一个组织。
如果用户已认证，则项目将添加到该用户的个人组织下。
如果用户未认证，项目将创建在一个新的 `unclaimed` 组织下。
然后可以通过项目的公共性能页面 `claimed`
或在后续调用 `bencher run` 时认证使用项目 slug 来 `claimed` 该组织。
