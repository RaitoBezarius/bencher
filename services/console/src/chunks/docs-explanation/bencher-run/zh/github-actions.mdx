### `--github-actions <GITHUB_TOKEN>`

<br />

可选：设置 GitHub API 身份认证令牌。最方便的方法是使用 [GitHub Actions `GITHUB_TOKEN` 环境变量][github token]（即 `--github-actions ${{ secrets.GITHUB_TOKEN }}`）。当这个选项设置好并且 `bencher run` 在 GitHub Actions 中作为一个拉取请求的一部分使用时，结果将作为一条评论添加到拉取请求中。这要求令牌具有 [具备 `write` 权限的 `pull-requests` 范围][github token permissions]。否则，结果将作为 GitHub 检查添加到提交中。这要求令牌具有 [具备 `write` 权限的 `checks` 范围][github token permissions]。无论哪种情况，结果也将被添加到作业摘要中。

[github token]: https://docs.github.com/zh/actions/security-guides/automatic-token-authentication
[github token permissions]: https://docs.github.com/zh/actions/security-for-github-actions/security-guides/automatic-token-authentication#permissions-for-the-github_token

> 🐰 如果您在 GitHub Action 中的 Docker 容器内运行，您将需要传入以下环境变量并挂载由 `GITHUB_EVENT_PATH` 指定的路径：
> - `GITHUB_ACTIONS`
> - `GITHUB_EVENT_NAME`
> - `GITHUB_EVENT_PATH`
> - `GITHUB_SHA`
> - `GITHUB_API_URL` (GitHub Enterprise Server 专用)
