### `--project <PROJECT>`

<br />

オプション: `--project` オプションまたは `BENCHER_PROJECT` 環境変数のいずれかを、プロジェクトのスラグまたはUUIDとして設定できます。指定された値がスラグで、プロジェクトがまだ存在しない場合は、自動的に作成されます。ただし、指定された値がUUIDの場合、プロジェクトはすでに存在している必要があります。両方が指定された場合、`--project` オプションが `BENCHER_PROJECT` 環境変数より優先されます。どちらも指定されない場合、次の方法でプロジェクトのスラグが生成されます:

1. 使用可能な場合、`git` リポジトリの親ディレクトリの名前。
2. 使用可能な場合、`git` リポジトリの初回コミットの7桁の16進短縮ハッシュ。
3. サポートされているオペレーティングシステムに基づく、ローカルマシンの13桁の英数字フィンガープリント。

例えば、生成されたプロジェクトのスラグは次のようになります: `project-abc4567-wxyz123456789`

新しいプロジェクトが作成された場合、スラグが指定されるか生成されるかに関わらず、プロジェクトは必ずいずれかの組織に所属しなければなりません。ユーザーが認証済みの場合、そのユーザーの個人用組織にプロジェクトが追加されます。ユーザーが認証されていない場合、プロジェクトは新しい `unclaimed` 組織の下に作成されます。この組織は、プロジェクトの公開されたPerfページから `claimed` するか、後の `bencher run` の呼び出しでユーザーが認証されている間にプロジェクトのスラグを使用して `claimed` できます。
