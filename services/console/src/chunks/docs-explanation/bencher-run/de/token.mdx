### `--token <TOKEN>`

<br />

Optional: Entweder die Option `--token` oder die Umgebungsvariable `BENCHER_API_TOKEN` kann auf ein gültiges API-Token gesetzt werden. Wenn beide angegeben sind, hat die Option `--token` Vorrang vor der Umgebungsvariable `BENCHER_API_TOKEN`. Wenn keiner angegeben ist, muss das Projekt `unclaimed` sein. Das heißt, wenn Sie bereits ein Projekt `claimed` haben, müssen Sie ein gültiges API-Token bereitstellen. [Klicken Sie hier, um ein API-Token zu erstellen](/console/users/tokens/add?back=L2RlL2RvY3MvZXhwbGFuYXRpb24vYmVuY2hlci1ydW4vIy0tdG9rZW4tdG9rZW4=).
