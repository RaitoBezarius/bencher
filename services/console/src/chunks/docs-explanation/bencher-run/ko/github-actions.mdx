### `--github-actions <GITHUB_TOKEN>`

<br />

선택 사항: GitHub API 인증 토큰을 설정하세요.
가장 편리한 방법은 [GitHub Actions `GITHUB_TOKEN` 환경 변수][github token]를 사용하는 것입니다(예: `--github-actions ${{ secrets.GITHUB_TOKEN }}`).
이 옵션이 설정되고 GitHub Actions에서 pull request의 일부로 `bencher run`이 사용될 경우,
결과가 pull request에 댓글로 추가됩니다.
이 경우 토큰에 [`write` 권한을 가진 `pull-requests` 범위가 필요합니다][github token permissions].
그렇지 않으면, 결과가 커밋에 GitHub Check로 추가됩니다.
이 경우 토큰에 [`write` 권한을 가진 `checks` 범위가 필요합니다][github token permissions].
어느 경우건, 결과는 작업 요약에도 추가됩니다.

[github token]: https://docs.github.com/ko/actions/security-guides/automatic-token-authentication
[github token permissions]: https://docs.github.com/ko/actions/security-for-github-actions/security-guides/automatic-token-authentication#permissions-for-the-github_token

> 🐰 GitHub Action 내의 Docker 컨테이너 안에서 실행하는 경우, 다음 환경 변수를 전달하고 `GITHUB_EVENT_PATH`로 지정된 경로를 마운트해야 합니다:
> - `GITHUB_ACTIONS`
> - `GITHUB_EVENT_NAME`
> - `GITHUB_EVENT_PATH`
> - `GITHUB_SHA`
