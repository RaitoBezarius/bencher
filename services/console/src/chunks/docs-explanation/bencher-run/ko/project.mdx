### `--project <PROJECT>`

<br />

선택 사항: `--project` 옵션이나 `BENCHER_PROJECT` 환경 변수 중 하나에 프로젝트의 슬러그 또는 UUID를 설정할 수 있습니다. 지정된 값이 슬러그이고 프로젝트가 아직 존재하지 않는 경우, 프로젝트가 자동으로 생성됩니다. 그러나 지정된 값이 UUID인 경우, 프로젝트가 이미 존재해야 합니다. 둘 다 지정된 경우, `--project` 옵션이 `BENCHER_PROJECT` 환경 변수보다 우선합니다. 둘 다 지정되지 않으면, 프로젝트 슬러그는 다음을 기준으로 자동 생성됩니다:

1. 사용 가능한 경우, `git` 저장소의 상위 디렉토리 이름.
2. 사용 가능한 경우, `git` 저장소의 초기 커밋에 대한 7자리 16진수 짧은 해시 값.
3. 지원되는 운영 체제의 경우, 로컬 머신의 13자리 영숫자 지문.

예를 들어, 생성된 프로젝트 슬러그는 다음과 같이 보일 수 있습니다: `project-abc4567-wxyz123456789`

새로운 프로젝트가 생성될 때, 슬러그가 지정되었든 생성되었든 간에, 프로젝트는 반드시 한 조직에 속해야 합니다. 사용자가 인증된 경우, 프로젝트는 해당 사용자의 개인 조직 하에 추가됩니다. 사용자가 인증되지 않은 경우, 프로젝트는 새로운 `unclaimed` 조직 하에 생성됩니다. 이 조직은 이후 프로젝트의 공개 성능 페이지에서 또는 인증된 상태로 `bencher run`을 재실행할 때 프로젝트 슬러그를 사용하여 `claimed`될 수 있습니다.
