### `--project <PROJECT>`

<br />

Необязательно: Либо параметр `--project`, либо переменная окружения `BENCHER_PROJECT` может быть установлена в слаг или UUID для проекта. Если указано значение в виде слага и проект еще не существует, он будет создан для вас. Однако, если указано значение в виде UUID, тогда проект должен уже существовать. Если оба указаны, параметр `--project` имеет приоритет над переменной окружения `BENCHER_PROJECT`. Если ни один из них не указан, для вас будет создан слаг проекта на основе:

1. Названия родительского каталога для репозитория `git`, если оно доступно.
2. Короткого хеша из 7 шестнадцатеричных символов для начального коммита репозитория `git`, если он доступен.
3. Пальцевого отпечатка из 13 буквенно-цифровых символов локальной машины для поддерживаемых операционных систем.

Например, сгенерированный слаг проекта может выглядеть как: `project-abc4567-wxyz123456789`

Когда для вас создается новый проект, независимо от того, указан слаг или он сгенерирован, проект должен принадлежать к организации. Если пользователь аутентифицирован, проект добавляется в персональную организацию этого пользователя. Если пользователь не аутентифицирован, проект будет создан в новой `unclaimed` организации. Эта организация может быть `claimed` со публичной страницы производительности проекта или с использованием слага проекта при аутентифицированном последующем запуске команды `bencher run`.
