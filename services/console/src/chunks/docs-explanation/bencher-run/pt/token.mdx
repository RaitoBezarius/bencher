### `--token <TOKEN>`

<br />

Opcional: A opção `--token` ou a variável de ambiente `BENCHER_API_TOKEN` podem ser configuradas com um token de API válido. Se ambos forem especificados, a opção `--token` prevalece sobre a variável de ambiente `BENCHER_API_TOKEN`. Se nenhum dos dois for especificado, o Projeto deve estar `não reivindicado`. Ou seja, se você já `reivindicou` um Projeto, deve fornecer um token de API válido. [Clique aqui para criar um token de API](/console/users/tokens/add?back=L3B0L2RvY3MvZXhwbGFuYXRpb24vYmVuY2hlci1ydW4vIy0tdG9rZW4tdG9rZW4=).
