### `--token <TOKEN>`

<br />

Optional: Either the `--token` option or the `BENCHER_API_TOKEN` environment variable
may be set to a valid API token.
If both are specified, the `--token` option takes precedence over the `BENCHER_API_TOKEN` environment variable.
If neither are specified then the Project must be `unclaimed`.
That is, if you have already `claimed` a Project you must provide a valid API token.
