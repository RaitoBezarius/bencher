### `--token <TOKEN>`

<br />

Opcional: Puede establecerse la opción `--token` o la variable de entorno `BENCHER_API_TOKEN` a un token de API válido.
Si ambos están especificados, la opción `--token` tiene prioridad sobre la variable de entorno `BENCHER_API_TOKEN`.
Si ninguno está especificado, el Proyecto debe estar `no reclamado`.
Es decir, si ya ha `reclamado` un Proyecto, debe proporcionar un token de API válido.
[Haga clic aquí para crear un token de API](/console/users/tokens/add?back=L2VzL2RvY3MvZXhwbGFuYXRpb24vYmVuY2hlci1ydW4vIy0tdG9rZW4tdG9rZW4=).
