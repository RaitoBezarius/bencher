### `--token <TOKEN>`

<br />

Optionnel : Soit l'option `--token`, soit la variable d'environnement `BENCHER_API_TOKEN`
peut être configurée avec un jeton API valide.
Si les deux sont spécifiés, l'option `--token` prévaut sur la variable d'environnement `BENCHER_API_TOKEN`.
Si aucun n'est spécifié, le Projet doit être `non réclamé`.
C'est-à-dire que si vous avez déjà `réclamé` un Projet, vous devez fournir un jeton API valide.
[Cliquez ici pour créer un jeton API](/console/users/tokens/add?back=L2ZyL2RvY3MvZXhwbGFuYXRpb24vYmVuY2hlci1ydW4vIy0tdG9rZW4tdG9rZW4=).
