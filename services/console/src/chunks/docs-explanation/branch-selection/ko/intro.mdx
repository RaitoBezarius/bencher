브랜치는 `git` 참조 (즉, 브랜치 이름이나 태그)입니다.
<code><a href="/ko/docs/explanation/bencher-run/">bencher run</a></code> CLI 서브커맨드를 사용할 때, `--branch` 옵션이나 `BENCHER_BRANCH` 환경 변수를 사용하여 브랜치를 선택하세요.
현재 `git` 해시는 `--hash` 옵션으로도 지정할 수 있습니다.
브랜치가 지정되지 않으면, 사용 가능한 경우 현재 `git` 브랜치 이름이 사용됩니다.
그렇지 않으면, 기본 브랜치로 `main`이 사용됩니다.
마찬가지로, `git` 해시가 지정되지 않으면, 사용 가능한 경우 현재 `git` 해시가 사용됩니다.

[지속적 벤치마킹][continuous benchmarking], 즉 CI에서의 벤치마킹 시, 종종 기존 브랜치에서 새로운 `branch`를 만들거나 리베이스하고 싶을 것입니다.
이것은 브랜치 [시작 지점][start point]이라고 합니다.
시작 지점은 특정 버전의 다른 브랜치(`git` 해시 사용 가능 시)입니다.
<code><a href="/ko/docs/explanation/bencher-run/">bencher run</a></code> CLI 서브커맨드를 사용할 때, `--start-point` 옵션으로 시작 지점을 선택하세요.
또한, `--start-point-hash` 옵션을 사용하여 시작 지점에 대한 특정 `git` 해시를 지정하세요.
시작 지점에서 복사되는 데이터의 양은 `--start-point-max-versions` 옵션으로 제어됩니다.
마찬가지로, [임계값][thresholds]은 `--start-point-clone-thresholds` 플래그를 사용하여 시작 지점에서 복사할 수 있습니다.
드리프트를 방지하기 위해, `--start-point-reset` 플래그를 사용하여 항상 브랜치를 시작 지점으로 재설정하세요.
