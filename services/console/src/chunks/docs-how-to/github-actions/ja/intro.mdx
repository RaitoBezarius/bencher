利用ケースに応じて、GitHub Actionsで[継続的ベンチマーク][continuous benchmarking]を次のためにセットアップできます。

- [ベースブランチ][base branch]
- [プルリクエスト][pull requests]
- [フォークからのプルリクエスト][fork pull requests]

続行する前に、[APIトークンを作成][create an api token]したうえで、**リポジトリ**のシークレットとして`BENCHER_API_TOKEN`という名前で設定してください。`Your Repo -> Settings -> Secrets and variables -> Actions -> New repository secret`に移動します。シークレット名を`BENCHER_API_TOKEN`として、シークレットの値にAPIトークンを設定します。

GitHub Actionsでは、[フォークされたリポジトリからワークフローがトリガーされた場合、シークレットはランナーに渡されません][github actions secrets]。したがって、以下のいずれかのワークフローをプルリクエストでリポジトリに追加する際には、_同じ_リポジトリのブランチを使用する必要があります。フォークからのプルリクエストでBencherを追加しようとすると、`BENCHER_API_TOKEN`シークレットは利用できません。`${{ secrets.BENCHER_API_TOKEN }}`は空の文字列になります。
