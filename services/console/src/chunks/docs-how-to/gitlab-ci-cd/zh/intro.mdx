根据您的使用案例，您可以在 GitLab CI/CD 中为您的以下项设置[持续基准测试][continuous benchmarking]：

- [目标分支](#target-branch)
- [合并请求](#merge-requests)

确保在继续之前已[创建 API token][created an api token]并将其设置为名为 `BENCHER_API_TOKEN` 的屏蔽变量！导航到 `Your Repo -> Settings -> CI/CD -> Variables -> Expand -> Add variable`。变量键应为 `BENCHER_API_TOKEN`，变量值应为您的 API token。勾选 `Protect variable` 和 `Mask variable` 这两个选项。
