## Projetos `unclaimed`

O Bencher permite que você acompanhe os resultados dos benchmarks sem criar uma conta. Quando você invoca o subcomando CLI <code><a href="/pt/docs/explanation/bencher-run/">bencher run</a></code> sem definir [a opção `--token`][token option], o Bencher criará um novo Projeto `unclaimed` [instantaneamente][on the fly projects] para seus resultados de benchmark. Se você seguiu os tutoriais de [Início Rápido][quick start] ou [Início Rápido Autônomo][self-hosted quick start], você estava aproveitando essa funcionalidade.

Qualquer pessoa pode visualizar e postar resultados de benchmark em um Projeto `unclaimed`. Para reivindicar um Projeto `unclaimed`, você precisará criar uma conta. Se você seguir o link de inscrição `Claim this project` no final da sua saída do <code><a href="/pt/docs/explanation/bencher-run/">bencher run</a></code>, você automaticamente reivindicará o Projeto quando sua conta for criada.

Se você já tem uma conta no Bencher, pode navegar até a [Página Pública de Desempenho][bencher perf] do Projeto e clicar no botão `Claim this project` no topo da página. Da mesma forma, se você já tem um token de API, pode usar o token de API com [a opção `--token`][token option] na próxima vez que você invocar o <code><a href="/pt/docs/explanation/bencher-run/">bencher run</a></code> para reivindicar o Projeto.

> 🐰 IMPORTANTE: Uma vez que um Projeto é `claimed`, ele não pode ser postado anonimamente. Você **deve** usar um token de API com [a opção `--token`][token option].
