A Branch는 `git` ref (즉, branch 이름 또는 태그)입니다.
<code><a href="/ko/docs/explanation/bencher-run/">bencher run</a></code> CLI 서브명령어를 사용할 때,
`--branch` 옵션 또는 `BENCHER_BRANCH` 환경 변수를 사용하여 Branch를 선택하세요.
기본적으로 모든 프로젝트는 `main` Branch로 시작하며,
둘 중 어느 것도 지정되지 않은 경우 기본적으로 `main`이 사용됩니다.

[지속적 벤치마킹][continuous benchmarking]을 할 때, 즉 CI에서 벤치마킹을 할 때,
기존에 존재하는 Branch에서 `branch`를 생성하거나 리베이스(rebase)하는 경우가 많습니다.
이를 Branch [시작 지점][start point]이라고 합니다.
Start Point는 특정 버전(가능하면 `git` 해시)에서의 다른 Branch입니다.
<code><a href="/ko/docs/explanation/bencher-run/">bencher run</a></code> CLI 서브명령어를 사용할 때,
`--branch-start-point` 옵션으로 Start Point를 선택하세요.
또한, `--branch-start-point-hash` 옵션을 사용하여 Start Point에 특정 `git` 해시를 지정할 수 있습니다.
모든 데이터와 [임계값][thresholds]은 Start Point에서 복사됩니다.
드리프트를 방지하려면 `--branch-reset` 플래그를 사용하여 항상 Branch를 Start Point로 재설정하세요.
