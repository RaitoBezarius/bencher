A Branch es la referencia `git` (es decir, el nombre de la rama o la etiqueta).
Al usar el subcomando CLI <code><a href="/es/docs/explanation/bencher-run/">bencher run</a></code>,
selecciona una Branch con la opción `--branch` o la variable de entorno `BENCHER_BRANCH`.
Por defecto, todos los proyectos comienzan con una Branch `main`, y
`main` se utiliza por defecto si no se especifica.

Cuando [Benchmarking Continuo][continuous benchmarking], es decir, benchmarking en CI,
a menudo querrá crear o rebasar una `branch` sobre una Branch ya existente.
Esto se llama un [Punto de Inicio de Branch][start point].
Un Punto de Inicio es otra Branch en una versión específica (y `git` hash, si está disponible).
Al usar el subcomando CLI <code><a href="/es/docs/explanation/bencher-run/">bencher run</a></code>,
selecciona un Punto de Inicio con la opción `--start-point`.
Además, usa la opción `--start-point-hash` para especificar un `git` hash específico para el Punto de Inicio.
La cantidad de datos que se copian desde el Punto de Inicio se controla con la opción `--start-point-max-versions`.
De manera similar, los [Umbrales][thresholds] pueden copiarse desde el Punto de Inicio con el flag `--start-point-clone-thresholds`.
Para prevenir desviaciones, utiliza el flag `--start-point-reset` para siempre restablecer la Branch al Punto de Inicio.
