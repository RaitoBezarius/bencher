Eine Branch ist die `git` Ref (d.h. Branch-Name oder Tag).
Beim Verwenden des <code><a href="/de/docs/explanation/bencher-run/">bencher run</a></code> CLI-Unterbefehls,
wähle eine Branch mit der Option `--branch` oder der Umgebungsvariable `BENCHER_BRANCH`.
Standardmäßig starten alle Projekte mit einer `main` Branch, und
`main` wird standardmäßig verwendet, falls weder noch angegeben sind.

Beim [Kontinuierlichen Benchmarking][continuous benchmarking], also Benchmarking in CI,
will man oft eine `branch` auf eine bereits existierende Branch erstellen oder rebasen.
Dies wird ein Branch [Startpunkt][start point] genannt.
Ein Startpunkt ist eine andere Branch zu einer bestimmten Version (und `git` Hash, falls verfügbar).
Beim Verwenden des <code><a href="/de/docs/explanation/bencher-run/">bencher run</a></code> CLI-Unterbefehls,
wähle einen Startpunkt mit der Option `--branch-start-point`.
Zusätzlich kann die Option `--branch-start-point-hash` verwendet werden, um einen bestimmten `git` Hash für den Startpunkt anzugeben.
Alle Daten und [Schwellenwerte][thresholds] werden vom Startpunkt kopiert.
Um Drift zu verhindern, benutze die Option `--branch-reset`, um die Branch immer auf den Startpunkt zurückzusetzen.
