A Branch é a referência (`git` ref, ou seja, nome da branch ou tag).
Ao usar o subcomando de CLI <code><a href="/pt/docs/explanation/bencher-run/">bencher run</a></code>,
selecione uma Branch com a opção `--branch` ou a variável de ambiente `BENCHER_BRANCH`.
Por padrão, todos os projetos começam com uma Branch `main`, e
`main` é usada por padrão se nenhuma das duas for especificada.

Ao fazer [Benchmarking Contínuo][continuous benchmarking], ou seja, benchmarking no CI,
você frequentemente desejará criar ou fazer rebase de uma `branch` em uma Branch já existente.
Isso é chamado de [Ponto de Partida][start point] da Branch.
Um Ponto de Partida é outra Branch em uma versão específica (e hash do `git`, se disponível).
Ao usar o subcomando de CLI <code><a href="/pt/docs/explanation/bencher-run/">bencher run</a></code>,
selecione um Ponto de Partida com a opção `--branch-start-point`.
Além disso, use a opção `--branch-start-point-hash` para especificar um hash específico do `git` para o Ponto de Partida.
Todos os dados e [Limites][thresholds] são copiados a partir do Ponto de Partida.
Para evitar desvios, use a flag `--branch-reset` para sempre resetar a Branch para o Ponto de Partida.
