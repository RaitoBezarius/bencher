### `--branch-reset`

<br />

Redefinir o `branch` especificado.
Quando a opção [`--branch-start-point`][branch start point] está configurada,
o `branch` será redefinido para o Ponto de Partida especificado.
Isso é útil para evitar desvio nos dados de [Benchmarking Contínuo Estatístico][statistical continuous benchmarking].
O principal caso de uso para `--branch-reset` sem especificar um Ponto de Partida
é o [Benchmarking Contínuo Relativo][relative continuous benchmarking].

- Se o `branch` não existir e nenhum Ponto de Partida for especificado, o `branch` será criado sem nenhum dado histórico.
- Se o `branch` existir e nenhum Ponto de Partida for especificado,
  a versão antiga do `branch` será tanto renomeada quanto arquivada,
  e um novo `branch` será criado sem nenhum dado histórico.
- Se o `branch` não existir e um Ponto de Partida for especificado,
  o `branch` será criado usando o Ponto de Partida.
- Se o `branch` existir e um Ponto de Partida for especificado,
  a versão antiga do `branch` será tanto renomeada quanto arquivada,
  e um novo `branch` será criado usando o Ponto de Partida.

Se um novo `branch` for criado com um Ponto de Partida,
então todos os dados e Limiares do Ponto de Partida especificado serão copiados.
Caso contrário, o `branch` será criado sem nenhum dado histórico.

Por exemplo, se `feature_branch` existir sem um Ponto de Partida,
então usar `--branch feature_branch --branch-reset`
faria:

1. Renomear o antigo `feature_branch` para `feature_branch@archive-suffix` e arquivá-lo
2. Criar um novo Branch chamado `feature_branch`
3. `bencher run` prosseguiria usando o novo `feature_branch`
