### `--github-actions <GITHUB_TOKEN>`

<br />

Opcional: Defina o token de autenticação da API do GitHub (ou seja, `--github-actions ${{ secrets.GITHUB_TOKEN }}`).
Quando esta opção é configurada e `bencher run` é utilizado no GitHub Actions como parte de um pull request,
os resultados serão adicionados ao pull request como um comentário.
Caso contrário, os resultados serão adicionados ao commit como um comentário de Verificação do GitHub.
A maneira mais conveniente de fazer isso é a [variável de ambiente `GITHUB_TOKEN` do GitHub Actions](https://docs.github.com/en/actions/security-guides/automatic-token-authentication).

> 🐰 Se você estiver executando dentro de um contêiner Docker no GitHub Action, será necessário passar as seguintes variáveis de ambiente e montar o caminho especificado por `GITHUB_EVENT_PATH`:
> - `GITHUB_ACTIONS`
> - `GITHUB_EVENT_NAME`
> - `GITHUB_EVENT_PATH`
